Pick & Place — PLC Control & Production Monitoring

Fully programmed Pick & Place system with structured PLC code and real-time cycle time monitoring — built from scratch using TIA Portal V17, PLCSIM (S7-1500), and Factory I/O.

Client Personal Project
Year 2026
Category PLC Programming · Production Monitoring
Role Automation engineer & system designer

The Problem

In manufacturing lines with pick-and-place robots, unplanned downtime is one of the most expensive problems. Cycle time degradation — when a machine gradually slows down — typically goes unnoticed until a full stop occurs. By the time maintenance is called, production has already been lost. Most plants lack real-time visibility into cycle-by-cycle performance.

The Solution

A complete, modular PLC program that controls a cartesian Pick & Place robot while simultaneously monitoring production KPIs in real time. The system measures every cycle, calculates statistical trends (average, minimum, maximum cycle times), and tracks machine states — building the foundation for early degradation detection before failures occur.

Architecture

FC1
Control — Safety & Master Logic (Ladder)

Start/Stop self-latching circuit, Emergency Stop handling with N.C. logic, fault detection and counting, forced output shutdown on stop — defense in depth

FC2
Sequence — State Machine (SCL)

10-step state machine with sub-steps for motion synchronization, two-phase motion pattern (wait for start → wait for end), automatic cycle repetition, conveyor coordination

FC3
Monitoring — Production Analytics (SCL)

Cycle time measurement via TON timer, statistical tracking (last, average, min, max), machine state classification, fault and piece counting

DB1
Data — Centralized Process Variables

All control states, production counters, cycle time statistics, and diagnostic data accessible from all function blocks

Key Technical Decisions

  • Two-phase motion pattern — Factory I/O axis signals are TRUE during motion, requiring a start-detection sub-step to prevent same-scan false transitions
  • Separate safety layer — FC1 handles all safety independently; if the sequence has a bug, safety still forces outputs OFF on emergency
  • TON as stopwatch — Used a timer on-delay with a long preset as a cycle chronometer instead of system clock reads — simpler and reliable in PLCSIM
  • Exit conveyor always running — Prevents product accumulation at output, addressing a real production concern

Monitored Variables

  • Cycle time per piece — last, average, minimum, maximum (ms)
  • Piece count — cumulative production counter
  • Machine state — Idle / Running / Fault classification
  • Fault count — accumulated emergency stops and errors
  • Current step — real-time sequence position for diagnostics

Results

  • Fully functional Pick & Place cycle running autonomously
  • Real-time cycle time measurement with statistical analysis
  • Clean, modular PLC code ready for industrial adaptation
  • Foundation for predictive maintenance via cycle time trend analysis
